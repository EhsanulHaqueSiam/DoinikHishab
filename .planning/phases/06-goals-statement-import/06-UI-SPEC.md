# Phase 6 — UI Design Contract

> Visual and interaction contract for Goals & Statement Import. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, checkbox/radio internal spacing |
| sm | 8px | Compact element spacing, amortization table cell padding |
| md | 16px | Default element spacing, card padding, section gaps, form field spacing |
| lg | 24px | Section padding, goal card internal layout |
| xl | 32px | Layout gaps between major sections (goal list to strategy comparison) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Import review list rows use 56px minimum height (matching SinkingFundRow minHeight pattern). Bottom sheet content uses 20px horizontal padding (matching AddSubscriptionForm px-5 pattern). Checkbox touch targets use 44px minimum (accessibility).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 10px (text-2xs) | 700 (bold) | 1.4 |
| Heading | 20px (text-xl) | 700 (bold) | 1.2 |
| Display | 36px (text-hero) | 700 (bold) | 1.17 |

Notes:
- Labels use uppercase + tracking-widest for section headers and form field labels (Input component pattern)
- Body text for list items uses text-sm font-bold for primary labels, text-2xs text-surface-800 for secondary info
- Amortization table headers use text-2xs font-bold uppercase tracking-wider text-surface-900
- Amortization table cells use text-2xs text-surface-800 for numeric values
- Goal card names use text-sm font-bold text-foreground (matching SinkingFundRow)
- Strategy delta row uses text-sm font-bold text-primary-700 for savings highlight

## Interaction Contracts

### Goal Management

| Interaction | Behavior |
|-------------|----------|
| Tap "Create Goal" | Open GoalForm bottom sheet (snap 70%) |
| Tap "Add Debt" | Open DebtForm bottom sheet (snap 70%) |
| Submit goal form | Validate name (required), amount (> 0), date (future). Save to MMKV. Close sheet. Haptic success |
| Submit debt form | Validate name (required), balance (> 0), APR (0-100), min payment (> 0). Save to MMKV. Close sheet. Haptic success |
| Tap goal card | Navigate to app/goals/[id].tsx detail screen |
| Tap debt card | Expand/collapse amortization table inline (no navigation) |
| Tap "Show schedule" | Toggle AmortizationTable visibility with layout animation (300ms) |
| Tap "Show more" in amortization | Load remaining rows below initial 6 (no pagination, just reveal) |
| Tap "Edit Goal" on detail | Re-open GoalForm pre-populated with current values |
| Tap "Delete Goal" on detail | Show confirmation alert: "Delete [name]? This cannot be undone." Two buttons: Cancel (default) / Delete (danger) |
| Tap GoalDashboardCard | Navigate to app/goals/index.tsx |
| Tap "See All" on dashboard card | Navigate to app/goals/index.tsx |

### Strategy Comparison

| Interaction | Behavior |
|-------------|----------|
| View with 2+ debts | Strategy comparison section auto-renders below debt list |
| View with 0-1 debts | Strategy comparison hidden. No empty state needed |
| Recommended strategy | Determined by lower total interest (always Avalanche). Highlighted with teal left border |

### Statement Import

| Interaction | Behavior |
|-------------|----------|
| Tap FilePickerButton | Open expo-document-picker filtered to PDF/XLS/TXT MIME types. Single file selection |
| File selected | Show ImportProgress (ActivityIndicator + "Parsing statement..."). Parse on-device. Replace picker with review list on completion |
| Parse failure | Show error state: "Couldn't parse this file. Make sure it's a valid bKash or Nagad statement." with "Try Another File" button |
| Tap checkbox on row | Toggle include/exclude for import. Update "Import N Transactions" count in CTA |
| Tap "Select All" | Check all non-duplicate rows. Duplicate rows remain unchecked |
| Tap "Deselect All" | Uncheck all rows |
| Tap type mapping pill | Cycle through: Expense -> Income -> Expense. No modal, inline toggle |
| Tap duplicate badge | No action (informational only). Row starts unchecked but user can override by checking |
| Tap "Import N Transactions" | Save selected transactions to MMKV. Show success toast (sonner-native): "Imported N transactions". Navigate back. Haptic success |
| Import with 0 selected | CTA disabled (opacity-30). Text: "Select transactions to import" |

### Navigation Integration

| Interaction | Behavior |
|-------------|----------|
| Dashboard "Goals" card tap | Navigate to app/goals/index.tsx |
| Settings "Import Statement" tap | Navigate to app/import/index.tsx |
| Transactions screen "Import" button tap | Navigate to app/import/index.tsx |
