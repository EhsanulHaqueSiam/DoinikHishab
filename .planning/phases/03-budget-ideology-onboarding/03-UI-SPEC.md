# Phase 3 -- UI Design Contract

> Visual and interaction contract for the Budget Ideology and Onboarding phase. This phase introduces sinking fund templates with progress visualization, the Ready to Assign hero banner, Age of Money and Days of Buffering dashboard metrics, contextual YNAB rule tips, and a 5-step guided onboarding flow with YNAB 4 Rules education. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, stepper dot spacing |
| sm | 8px | Compact element spacing, progress bar text gap, rule tip icon margin |
| md | 16px | Default element spacing, card padding, carousel horizontal margin |
| lg | 24px | Section padding, sinking fund section margin, onboarding step content padding |
| xl | 32px | Layout gaps, carousel card horizontal padding |
| 2xl | 48px | Major section breaks, onboarding illustration height spacer |
| 3xl | 64px | Page-level spacing, onboarding header top padding |

Exceptions:
- Stepper dots: 12px diameter (3 x 4px), active dot 24px width. Not on 8-point scale but 4-point aligned.
- Carousel card snap interval: capped at 448px max (480px WebContainer - 32px horizontal margin).

**Source:** Phase 1 UI-SPEC spacing scale (unchanged), new exceptions for Phase 3 components.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class | Phase 3 Usage |
|------|------|--------|-------------|----------------|---------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Rule card descriptions, onboarding step body text, auto-suggest text |
| Label | 10px | 600 (semibold) | 1.4 | `text-2xs font-semibold` | Section headers ("True Expenses"), metric labels ("AGE OF MONEY"), stepper labels, status badges |
| Heading | 18px | 700 (bold) | 1.2 | `text-lg font-bold` | Rule card titles, onboarding step headings, sinking fund names |
| Hero | 36px | 700 (bold) | 1.1 | `text-hero font-bold` | Ready to Assign amount |

Weights declared: 400 (regular) and 700 (bold).

**Weight rationale:** The existing codebase uses `font-bold` extensively (BalanceCard hero, BudgetRow amounts, GoalProgress percentage, BudgetScreen group headers). Phase 1 UI-SPEC recommended 600 (semibold) but the actual codebase implementations use `font-bold` (700). This contract aligns with the codebase reality. Two weights: 400 for body/description text, 700 for headings, amounts, and emphasis.

**Bengali text note:** SpaceMono lacks Bengali glyphs. System falls back to device default (Noto Sans Bengali on Android, Kohinoor Bangla on iOS). Bengali strings are approximately 1.3x longer than English equivalents. All text containers must use flex layouts, not fixed widths. Apply `numberOfLines={1}` with `ellipsizeMode="tail"` for single-line constrained labels (sinking fund names in progress rows). Multi-line text (rule descriptions, onboarding body) must not be line-clamped.

**Source:** tailwind.config.js fontSize, existing component class analysis (BalanceCard, BudgetRow, GoalProgress).

1. **Month Navigator** -- existing, unchanged. `bg-surface-100`, `border-b border-border`.

## Interaction Contracts

### Ready to Assign Hero

- **Tap**: Navigate to assignment view (existing AssignMoney modal or future dedicated screen)
- **Touch target**: Full card width, minimum 64px height (hero text + subtitle + padding)
- **Feedback**: `active:bg-surface-400/30` on Pressable

### Sinking Fund Row

- **Tap**: Navigate to sinking fund detail (`/budget/{categoryId}`)
- **Touch target**: Full row width, minimum 56px height (name + progress bar + suggest text)
- **Feedback**: `active:bg-surface-400/30` on Pressable

### Rule Carousel

- **Swipe**: Horizontal swipe advances to next/previous rule card
- **Snap**: Cards snap to nearest card boundary (`snapToInterval`)
- **Pagination dots**: Update on scroll via `onScroll` event, `scrollEventThrottle={16}`
- **No autoplay**: Cards do not auto-advance

### Stepper Dots

- **Tap**: Tapping a completed dot navigates back to that step
- **Touch target**: Each dot wrapped in Pressable with `hitSlop={12}` for minimum 36px tap area
- **No forward skip via dots**: Cannot tap a future incomplete dot to skip ahead (use "Skip Setup" button instead)

### Rule Tip Dismiss

- **Tap "Got it"**: Dismiss tip permanently (MMKV persist)
- **Touch target**: "Got it" text with `hitSlop={8}`, minimum 32px tap area
- **Animation**: Tip collapses with 200ms `withTiming` height animation via Reanimated

### MetricsCard Settings Gear

- **Tap**: Opens bottom sheet to select lookback period (30 / 60 / 90 / 180 days)
- **Touch target**: Settings gear icon 12px with `hitSlop={8}` for 28px tap area
- **Feedback**: `active:opacity-60`
