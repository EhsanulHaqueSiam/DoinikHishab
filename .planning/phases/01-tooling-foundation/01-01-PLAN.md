<task type="auto">
  <name>Task 2: Set up lefthook pre-commit hook and add CI lint/test jobs</name>
  <files>lefthook.yml, .github/workflows/build.yml</files>
  <read_first>
    - .github/workflows/build.yml (current CI workflow structure, jobs, env vars)

    - .planning/phases/01-tooling-foundation/01-CONTEXT.md (D-02: CI + pre-commit, D-03: lefthook not husky)
  </read_first>
  <action>
1. Create `lefthook.yml` at project root with this exact configuration (per D-03, using lefthook not husky/lint-staged):
   ```yaml
   pre-commit:
     commands:
       biome:
         glob: "*.{js,ts,cjs,mjs,jsx,tsx,json,jsonc}"
         run: bunx @biomejs/biome check --write --no-errors-on-unmatched --files-ignore-unknown=true --colors=off {staged_files}
         stage_fixed: true
   ```
   Key details:
   - `stage_fixed: true` re-stages files after auto-formatting (so the commit includes the formatted version)
   - `--no-errors-on-unmatched` prevents errors if no matching files are staged
   - `--files-ignore-unknown=true` skips non-JS files gracefully
   - `--colors=off` for cleaner terminal output in hooks

2. Install lefthook. Try system install first (Arch Linux):
   ```
   which lefthook || sudo pacman -S lefthook 2>/dev/null || bun add -d lefthook
   ```
   If system install not available, fall back to npm dev dependency per RESEARCH.md fallback strategy.

3. Run `lefthook install` to register the git hook in `.git/hooks/pre-commit`.

4. Add a new `lint` job to `.github/workflows/build.yml` that runs BEFORE the `typecheck`, `web`, and `android` jobs. Insert this job between the env block and the existing `convex-deploy` job:
   ```yaml
   lint:
     runs-on: ubuntu-latest
     steps:
       - uses: actions/checkout@v4

       - uses: oven-sh/setup-bun@v2
         with:
           bun-version: ${{ env.BUN_VERSION }}

       - name: Install dependencies
         run: bun install --frozen-lockfile || bun install

       - name: Lint check
         run: bun run lint
   ```

5. Make the `web` and `android` jobs depend on `lint` by adding `needs: [lint]` to their job definitions. The `typecheck` job should also get `needs: [lint]` so lint runs first and blocks if it fails (per D-02 belt-and-suspenders enforcement).

6. Verify lefthook is active by checking `.git/hooks/pre-commit` contains lefthook reference.
